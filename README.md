# 인플루언서 인게이지먼트 핏 분석 대시보드

인플루언서 컨택 전에, 브랜드와 잘 맞는 인스타그램 계정을 데이터 기반으로 골라내는 도구입니다.

## 어떻게 동작하나요

1. **로컬 PC에서 수집** — `scraper/collect.py` 가 Playwright로 **본인 Chrome 로그인 세션**을
   재사용해 인스타그램 프로필 페이지를 실제로 방문하고, 그 페이지가 자체적으로 호출하는
   공개 프로필 API 응답(팔로워 수, 최근 게시물 좋아요/댓글 수, 소개글 등)을 가로채 저장합니다.
2. **GitHub에 푸시** — `PUSH-Update.bat` 을 실행하면 수집한 데이터가 GitHub 저장소에 올라갑니다.
3. **Streamlit Cloud에서 조회** — 클라우드 대시보드는 이 CSV를 읽어서 순위표·차트·상세 프로필을 보여줍니다.

수집은 항상 로컬에서, 조회는 클라우드에서 — 기존 META 광고 레퍼런스 대시보드와 동일한 구조입니다.

### 왜 로그인이 필요한가요

원래는 로그인 없이 공개 API만 호출하는 방식으로 만들었지만, 실제 테스트 결과 인스타그램이
비로그인 요청은 거의 항상 즉시 차단(429)한다는 걸 확인했습니다. 일반 프로필 페이지(HTML)는
열리지만 실제 데이터 없는 빈 껍데기만 내려주고, JS 렌더링 + 로그인 세션이 있어야 실제
콘텐츠가 채워지는 구조였습니다. 그래서 본인 계정으로 로그인된 브라우저 세션을 재사용하는
방식으로 전환했습니다 (사용자 승인 후 진행).

## ⚠️ 꼭 읽어주세요 — 계정 위험 관련 주의사항

- **본인 인스타그램 계정으로 로그인된 세션**을 사용합니다. 자동화가 탐지되면 계정이 일시
  제한되거나 정지될 실질적인 위험이 있습니다 — 이 위험을 감수하고 사용하는 도구입니다.
- **하루 수십 계정 이내로, 소량으로만** 사용하세요. 계정 사이 10~25초 지연이 기본 적용됩니다
  (`config.json` 의 `request_delay_seconds` 로 조정 가능 — 줄이지 말고 오히려 늘리는 걸 권장).
- 로그인/보안 확인(체크포인트) 페이지로 리다이렉트되면 **수집이 자동으로 즉시 중단**됩니다.
  이 경우 실행을 멈추고, 본인 스마트폰 인스타그램 앱을 열어 정상 로그인이 유지되는지 먼저
  확인하세요. 며칠간 수집을 쉬는 걸 권장합니다.
- 절대 다량의 계정을 한 번에 수집하거나, 짧은 간격으로 반복 실행하지 마세요.
- 비공개(private) 계정은 팔로워 수 외의 정보(게시물 좋아요/댓글)를 가져올 수 없습니다.
- 수집 중에는 실제 Chrome 창이 화면에 뜹니다 (headless=False — 탐지 회피 목적). 다른 작업을
  방해할 수 있으니 창을 최소화하지 말고 그대로 두세요.
- `config.json` 의 `chrome_profile_name` 은 인스타그램에 로그인되어 있는 프로필 이름으로
  맞춰야 합니다 (Chrome 우상단 프로필 아이콘에서 확인 가능).
- 대시보드의 "계정 추가" 기능은 **로컬 실행 시에만** 동작합니다. Streamlit Cloud에서는 조회만
  가능하도록 막혀 있습니다 (클라우드 IP는 더 쉽게 차단되고, 로그인 세션을 클라우드에 올리는
  것 자체가 더 위험하기 때문).

## 로컬에서 실행하기

1. `START-Windows.bat` 더블클릭 — 최초 실행 시 자동으로 필요한 패키지를 설치합니다.
2. `config.json` 의 `influencers` 목록에 확인하고 싶은 인스타그램 아이디를 추가하세요.
   (또는 대시보드 사이드바에서 "계정 추가"로 하나씩 추가해도 됩니다.)
3. `COLLECT-Windows.bat` 을 실행해 수집합니다.
4. 대시보드에서 결과를 확인하세요.
5. 클라우드에도 반영하려면 `PUSH-Update.bat` 을 실행하세요.

## 핏 스코어는 어떻게 계산되나요

`dashboard/scoring.py` 에서 계산하며, 아래 네 가지를 종합합니다.

| 기준 | 비중 | 설명 |
|---|---|---|
| 인게이지먼트율 | 50% | (평균 좋아요+댓글) / 팔로워 — 최근 게시물 최대 12개 기준 |
| 브랜드 키워드 매칭 | 25% | 소개글·카테고리에 브랜드 키워드가 있으면 가산 |
| 게시 활발도 | 15% | 최근 게시물 간 평균 업로드 간격 |
| 팔로워 구간 적합 | 10% | 사이드바에서 선택한 타겟 구간(나노/마이크로/매크로/메가)과 일치하는지 |

브랜드 키워드는 대시보드 사이드바에서 바로 수정할 수 있고, `config.json` 에 저장됩니다.

## 파일 구조

```
config.json                  브랜드 키워드, 수집 대상 계정 목록, Chrome 프로필 설정, 요청 지연 시간
scraper/browser.py           본인 Chrome 프로필로 Playwright 브라우저 세션 실행
scraper/human_like.py        지연/스크롤/마우스 등 사람형 행동 시뮬레이션
scraper/instagram_browser.py 프로필 페이지 방문 + 네트워크 응답 캡처
scraper/instagram_public.py  캡처한 응답을 정리된 필드로 파싱
scraper/collect.py           수집 실행 스크립트 (CLI)
dashboard/scoring.py         핏 스코어 계산 로직
dashboard/app.py             Streamlit 대시보드 화면
data/influencers.csv         수집 결과 (자동 생성)
```
